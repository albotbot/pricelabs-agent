1. `pricelabs_set_overrides` in overrides.ts (line ~120, the async handler)
2. `pricelabs_delete_overrides` in overrides.ts (line ~320, the async handler)
3. `pricelabs_update_listings` in listings.ts (line ~158, the async handler)

For each write tool handler, add this check as the very first line inside the async callback, BEFORE any validation or API call:

```typescript
// --- Write safety gate (SAFE-01) ---
const writesEnabled = process.env.PRICELABS_WRITES_ENABLED;
if (writesEnabled !== "true") {
  return {
    content: [
      {
        type: "text" as const,
        text: "Write operations are disabled. Set PRICELABS_WRITES_ENABLED=true to enable.",
      },
    ],
    isError: true,
  };
}
```

IMPORTANT:
- Check `!== "true"` (not truthy check) so that unset, "false", "0", or any other value blocks writes. Only the exact string "true" enables writes.
- Do NOT change tool registration, descriptions, schemas, or annotations. The tools must remain visible in tools/list.
- Do NOT add a startup-time check or console log. The gate is per-call, checked when the tool is invoked.
- The existing tool logic (validation, API calls, cache invalidation) remains entirely unchanged after the gate check.

In listings.ts, the `pricelabs_update_listings` tool uses `server.registerTool()` (not `server.tool()`), so the handler is `async (params) => { ... }`. Add the gate check as the first thing inside that handler, before the try/catch block.

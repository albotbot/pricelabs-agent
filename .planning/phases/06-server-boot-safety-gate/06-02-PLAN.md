  <action>
Create two files: a Node.js validation script and a shell wrapper.

**scripts/validate-boot.mjs** (the main script, ESM for top-level await):

Spawns the MCP server as a child process, communicates via stdin/stdout JSON-RPC, and validates all 4 Phase 6 requirements:

**BOOT-01: Build and Start**
- Run `npm run build` in `mcp-servers/pricelabs/` and verify exit code 0
- Start `node dist/index.js` with a dummy `PRICELABS_API_KEY=test-key-for-validation` and a temp database path
- The server uses stdio transport, so it starts and waits for JSON-RPC on stdin. Send an `initialize` request (protocolVersion: "2024-11-05", capabilities: {}, clientInfo: {name: "boot-validator", version: "1.0.0"}) and verify it responds with a result
- Follow up with a `notifications/initialized` notification
- Use a 10-second timeout -- if the process doesn't respond, fail

**BOOT-02: Database Creation**
- After initialize succeeds, check that the SQLite database file exists at the temp path using `existsSync`
- Use `sqlite3 $DB_PATH ".tables"` to list tables (handle missing sqlite3 CLI gracefully)
- Verify all 7 tables exist: listing_snapshots, price_snapshots, reservations, market_snapshots, audit_log, change_tracking, user_config

**BOOT-03: Tool Registration**
- Send a `tools/list` JSON-RPC request to the running server via stdin
- Parse the response and count the number of tools returned
- Verify count equals 28
- Print all tool names sorted alphabetically for visibility

**SAFE-01: Write Tools Disabled**
- Send `tools/call` requests for each of the 3 write tools with minimal valid arguments:
  - pricelabs_set_overrides: {listing_id: "test", pms: "test", overrides: [], reason: "test"}
  - pricelabs_delete_overrides: {listing_id: "test", pms: "test", dates: ["2025-01-01"], reason: "test"}
  - pricelabs_update_listings: {listings: [{listing_id: "test", pms: "test"}], reason: "test"}
- For each, verify the response text contains "Write operations are disabled" and isError is true
- The server must NOT have PRICELABS_WRITES_ENABLED=true set during this test

Implementation details:
- Use colored output: green PASS / red FAIL with ANSI codes
- Track failures counter, exit with code 1 if any failures
- Send JSON-RPC as newline-delimited JSON (MCP stdio transport protocol)
- Clean up by sending SIGTERM to server process in finally block
- Create temp database in os.tmpdir() via mkdtempSync

**scripts/validate-boot.sh** (thin shell wrapper):
```bash
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
node scripts/validate-boot.mjs
```

Make the shell script executable with `chmod +x`.

IMPORTANT: The .mjs extension is required for top-level await without a package.json type:module in the scripts directory.

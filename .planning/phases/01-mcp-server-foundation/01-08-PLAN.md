  <action>
Create the main entry point following the pattern from research (Complete MCP Server Entry Point).

**Environment validation:**
```typescript
const apiKey = process.env.PRICELABS_API_KEY;
if (!apiKey) {
  console.error("FATAL: PRICELABS_API_KEY environment variable is required. " +
    "Set it in openclaw.json -> agents.list[].mcp.servers[].env");
  process.exit(1);
}
```
Exit with a clear, actionable error if the API key is missing. NEVER log the actual key value (INFRA-04).

**Service initialization:**
```typescript
const baseUrl = process.env.PRICELABS_BASE_URL || "https://api.pricelabs.co";
const rateLimiter = new TokenBucketRateLimiter(1000, 3600000);
const cache = new TtlCache();
const apiClient = new PriceLabsApiClient(apiKey, baseUrl, rateLimiter);
```

**Server creation:**
```typescript
const server = new McpServer({
  name: "pricelabs",
  version: "1.0.0",
});
```

**Tool registration -- import and call all 8 registration functions:**
```typescript
registerListingTools(server, apiClient, cache, rateLimiter);
registerPriceTools(server, apiClient, cache, rateLimiter);
registerOverrideTools(server, apiClient, cache, rateLimiter);
registerNeighborhoodTools(server, apiClient, cache, rateLimiter);
registerReservationTools(server, apiClient, cache, rateLimiter);
registerSyncTools(server, apiClient, cache, rateLimiter);
registerRatePlanTools(server, apiClient, cache, rateLimiter);
registerStatusTools(server, rateLimiter, cache);
```

**Transport connection:**
```typescript
const transport = new StdioServerTransport();
await server.connect(transport);
```

**Process error handling:**
```typescript
process.on("uncaughtException", (error) => {
  console.error("Uncaught exception:", error.message);
  process.exit(1);
});
process.on("unhandledRejection", (reason) => {
  console.error("Unhandled rejection:", reason);
  process.exit(1);
});
```

The entire file should use top-level await (supported by ES2022 + NodeNext modules). All imports use `.js` extensions for ESM compatibility (e.g., `import { ... } from "./services/rate-limiter.js"`).

After wiring, build the project: `npx tsc` to compile to dist/.

**Gateway security (INFRA-05, Pitfall 5):**
- `gateway.mode`: "local"
- `gateway.bind`: "loopback" -- CRITICAL: prevents network exposure (CVE-2026-25253)
- `gateway.auth.mode`: "token" -- requires authentication
- `gateway.auth.token`: "${OPENCLAW_GATEWAY_TOKEN}" -- reads from .env

**Agent configuration with MCP server:**
- `agents.list[0].id`: "pricelabs"
- `agents.list[0].name`: "PriceLabs Revenue Agent"
- `agents.list[0].model`: "anthropic/claude-opus-4-6"
- `agents.list[0].mcp.servers[0]`:
  - name: "pricelabs"
  - command: "node"
  - args: path to compiled dist/index.js (use relative path from workspace)
  - env.PRICELABS_API_KEY: "${PRICELABS_API_KEY}" -- reads from .env, NEVER hardcoded (INFRA-04)
  - env.PRICELABS_BASE_URL: "https://api.pricelabs.co"

**Docker sandbox (Pitfall 7):**
- `agents.defaults.sandbox.mode`: "all"
- `agents.defaults.sandbox.scope`: "agent"
- `agents.defaults.sandbox.workspaceAccess`: "ro" (read-only workspace access)

**Tool deny list (defense in depth):**
- Deny: group:automation, group:runtime, exec, write, edit, apply_patch, process, sessions_spawn, sessions_send, gateway, cron
- fs.workspaceOnly: true
- exec.security: "deny"
- elevated.enabled: false

**Channel security (INFRA-05):**
- Slack: enabled, socket mode, dmPolicy "pairing", threadReply true
- Telegram: enabled, dmPolicy "pairing", groups require mention
- Tokens reference env vars: ${SLACK_APP_TOKEN}, ${SLACK_BOT_TOKEN}, ${TELEGRAM_BOT_TOKEN}

**Logging with redaction (INFRA-04):**
- redactSensitive: "tools"
- redactPatterns: ["X-API-Key", "api[_-]?key", "PRICELABS"]

**Session config:**
- dmScope: "per-channel-peer"

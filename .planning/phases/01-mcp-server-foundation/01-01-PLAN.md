<task type="auto">
  <name>Task 2: Implement token bucket rate limiter</name>
  <files>mcp-servers/pricelabs/src/services/rate-limiter.ts</files>
  <action>
Implement `TokenBucketRateLimiter` class following the pattern from research (Pattern 2). This is a timer-free implementation -- tokens are calculated lazily on each call based on elapsed time.

**Constructor:**
- `maxTokens: number = 1000` (PriceLabs 1000/hr limit)
- `refillWindowMs: number = 3600000` (1 hour in ms)
- Initialize `tokens = maxTokens`, `lastRefillTime = Date.now()`
- Calculate `refillRate = maxTokens / refillWindowMs` (tokens per ms)

**Methods:**
- `private refill(): void` -- calculates elapsed time since last refill, adds proportional tokens, caps at maxTokens, updates lastRefillTime
- `tryConsume(count: number = 1): { allowed: boolean; retryAfterMs?: number }` -- calls refill(), checks if enough tokens, deducts if so, returns retry estimate if not
- `getStatus(): { remaining: number; max: number; resetMs: number; utilizationPct: number }` -- calls refill(), returns current state including utilization percentage (for get_api_status tool)

**Important:** Do NOT use setInterval or any timer. The lazy refill approach is simpler, testable, and avoids timer drift. Export the class as a named export.
  </action>

Run `npx tsc --noEmit` from `mcp-servers/pricelabs/` -- no type errors.
Manually verify: the class has tryConsume, getStatus, and private refill methods.

**cache.ts -- TtlCache class** (Pattern 3 from research):

Interface `CacheEntry<T>`: data, cachedAt (timestamp), ttlMs.

Methods:
- `get<T>(key: string): { data: T; cacheAgeSeconds: number } | null` -- returns null if not found or expired (delete expired on access). Includes `cacheAgeSeconds` for response metadata (locked decision: every response includes cache_age_seconds).
- `set<T>(key: string, data: T, ttlMs: number): void` -- stores with timestamp.
- `invalidate(pattern: string): void` -- deletes all entries whose key starts with `pattern`. Used by write tools to invalidate affected cache entries.
- `getStats(): { entries: number; oldestEntryAgeSeconds: number }` -- for get_api_status tool. Count active (non-expired) entries and find oldest.
- `clear(): void` -- remove all entries.

Track hit/miss counts as private fields (`hits: number = 0`, `misses: number = 0`) incremented in `get()`. Expose via `getStats()` as `hitRate` (string percentage like "73.2%").

**errors.ts -- Error classification:**

Three custom error classes extending Error:
- `RateLimitError` -- has `retryAfterMs: number` property. Thrown when token bucket denies a request.
- `AuthError` -- thrown on 401 responses. Critical severity (locked decision: immediate alert).
- `ApiError` -- has `statusCode: number` and `body: string` properties. For non-auth API failures (400, 404, 500, etc.).

Helper function `isRetryable(error: unknown): boolean` -- returns true for ApiError with status 429, 500, 502, 503, 504. Returns false for 400, 401, 403, 404. Returns false for AuthError (never retry auth).

Export all classes and the helper function as named exports.

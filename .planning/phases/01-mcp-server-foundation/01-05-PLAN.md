  <files>mcp-servers/pricelabs/src/tools/listings.ts</files>
  <action>
Create `registerListingTools` function that takes `(server: McpServer, apiClient: PriceLabsApiClient, cache: TtlCache, rateLimiter: TokenBucketRateLimiter)` and registers the listing tools.

**Tool: pricelabs_get_listings**
- Description: "Fetch all PriceLabs listings with health scores, occupancy, revenue, and sync status. Returns computed fields: occupancy_gap_pct, revenue_vs_stly_pct, days_since_sync."
- Input schema: GetListingsInputSchema (skip_hidden?: boolean, only_syncing?: boolean)
- Annotations: readOnlyHint: true, destructiveHint: false, openWorldHint: true
- Cache key: `listings:all:${JSON.stringify(params)}` with TTL 60 minutes (3600000ms)
- Handler:
  1. Use fetchWithFallback with fetcher that calls `apiClient.get("/v1/listings")` with query params
  2. Pass `computeListingFields` to compute occupancy_gap_pct, revenue_vs_stly_pct, days_since_sync for each listing
  3. For array responses: compute fields per-listing, return as array in computed.listings_computed
  4. Return JSON.stringify of ToolResponse as text content

**Tool: pricelabs_get_listing**
- Description: "Fetch a single PriceLabs listing by ID with detailed health, occupancy, revenue data, and computed analysis fields."
- Input schema: GetListingInputSchema (listing_id: string, pms: string)
- Annotations: readOnlyHint: true, destructiveHint: false, openWorldHint: true
- Cache key: `listing:${params.listing_id}:${params.pms}` with TTL 60 minutes
- Handler: Same pattern as get_listings but for single listing endpoint

For tool registration, check if `server.tool()` supports an annotations parameter. If not, use `server.registerTool()` which accepts annotations in the tool definition object (see research Open Question 1). At implementation time, inspect the SDK types to determine the correct overload. If neither supports annotations cleanly, register tools with `server.tool()` and add a comment noting that annotations should be added when SDK support is confirmed.

Error handling: Wrap each handler in try/catch. On error, return MCP error response format: `{ content: [{ type: "text", text: errorMessage }], isError: true }`. Error messages must be plain and direct (locked decision), include what happened and suggest next steps. For auth errors, include critical severity note.

  <name>Task 2: Register write listing tool (update_listings) with audit trail</name>
  <files>mcp-servers/pricelabs/src/tools/listings.ts</files>
  <action>
Add the write tool to the existing registerListingTools function.

**Tool: pricelabs_update_listings**
- Description: "Update base/min/max prices or tags for one or more listings. REQUIRES reason parameter for audit trail. This is a DESTRUCTIVE operation -- changes take effect immediately."
- Input schema: UpdateListingsInputSchema (listings: array of {id, pms, min?, base?, max?, tags?}, reason: string min 10 chars)
- Annotations: readOnlyHint: false, destructiveHint: true, openWorldHint: true
- Handler:
  1. Log the reason parameter (this creates the audit trail -- the MCP response includes the reason in its output)
  2. Call `apiClient.post("/v1/listings", { listings: params.listings })`
  3. After successful write, invalidate listing cache entries: `cache.invalidate("listings:")` and `cache.invalidate("listing:")` -- all listing cache is stale after update
  4. Return the API response with meta showing data_source: "live", cache_age_seconds: 0
  5. Include the reason in the response so the agent can reference it in conversations

No fetchWithFallback for write operations -- writes must go to the live API. If the API is down or rate limited, return an error (do not serve cached data for writes).

Write-specific error handling:
- RateLimitError: "Cannot update listings -- rate limit reached. No cached data is used for write operations. Try again in X minutes."
- AuthError: Critical -- "Authentication failed. Listing update could not be executed. Check PRICELABS_API_KEY immediately."
- ApiError: Include status code and suggest checking listing IDs/PMS values

**Tool: pricelabs_get_overrides**

**Tool: pricelabs_set_overrides** (CRITICAL -- highest-risk write tool)
- Description: "Create or update date-specific overrides (DSOs) for a listing. DSOs have the HIGHEST priority in PriceLabs and override ALL other settings including min price. REQUIRES reason for audit trail."
- Input: SetOverridesInputSchema (listing_id, pms, overrides array, reason)
- Annotations: destructiveHint: true, readOnlyHint: false

Pre-write validation (implements Pitfalls 1, 2, 3 from research):

Step 1 -- Validate percentage range: For each override with price_type "percentage", verify price_value is in range -75 to 500. Reject entire batch if any violate. (This is also in the Zod schema, but double-check here for safety.)

Step 2 -- Validate currency for fixed prices (Pitfall 3): For each override with price_type "fixed", verify the `currency` field is present. Then fetch the listing's current data to confirm currency matches the PMS listing currency. If mismatch, reject with error: "Fixed-price DSO currency ({given}) does not match listing currency ({actual}). This would apply the wrong price."

Step 3 -- Validate price floor (Pitfall 1): For each override with price_type "fixed", fetch the listing's min_price. If the DSO fixed price is below min_price, reject with error: "DSO fixed price (${dso_price}) is below listing minimum price (${min_price}). DSOs override min price -- this would push a below-minimum rate to the OTA."

Step 4 -- Execute write: Call `apiClient.post("/v1/listings/${listing_id}/overrides", { overrides })`.

Step 5 -- Post-write verification (Pitfall 2): Immediately GET overrides for the same listing/date range. Compare the number of dates in the request vs the response. If response has fewer dates, report: "Warning: ${missing_count} dates were silently dropped by PriceLabs. Dates not confirmed: [list]. This usually means those dates are in the past or outside the sync window."

Step 6 -- Invalidate cache: `cache.invalidate("overrides:" + listing_id)`.

Step 7 -- Return response with write result, verification status, and the reason.

**Tool: pricelabs_delete_overrides**
- Description: "Delete date-specific overrides for specified dates. Removes DSO pricing and restores dynamic pricing for those dates. REQUIRES reason for audit trail."
- Input: DeleteOverridesInputSchema (listing_id, pms, dates, reason)
- Annotations: destructiveHint: true
- Handler:
  1. Call `apiClient.delete("/v1/listings/${listing_id}/overrides")` with dates in body/params (check API spec for exact format)
  2. Invalidate override cache for this listing
  3. Return confirmation with reason

All write tools: Do NOT use fetchWithFallback. Writes must go live. If rate limited, return error explaining the write cannot proceed.

Implement `PriceLabsApiClient` class following Pattern from research (API Client section).

**Constructor:**
- `apiKey: string` -- stored privately, NEVER logged or included in error messages
- `baseUrl: string` -- defaults to "https://api.pricelabs.co"
- `rateLimiter: TokenBucketRateLimiter` -- injected dependency
- `timeoutMs: number = 300000` -- PriceLabs recommended 300-second timeout

Build headers object with `X-API-Key` and `Content-Type: application/json`.

**Core method -- `request<T>(method, path, body?, retryCount?): Promise<ApiResponse<T>>`:**

1. Check rate limiter via `rateLimiter.tryConsume()`. If not allowed, throw `RateLimitError(retryAfterMs)`.
2. Create AbortController with timeout (clearTimeout in finally block).
3. Call global `fetch()` (Node 22 built-in, no library needed).
4. Handle response statuses:
   - **429**: If `retryCount < 3`, wait per `Retry-After` header (or 60s default), then recursively retry. If max retries exceeded, throw `RateLimitError`.
   - **401**: Immediately throw `AuthError` -- NEVER retry auth failures (locked decision: critical severity). Error message must NOT include the API key value.
   - **500/502/503/504**: If `retryCount < 3`, wait with exponential backoff + jitter (base 1s, max 30s), then retry. If exhausted, throw `ApiError(status, body)`.
   - **400/403/404**: Throw `ApiError(status, body)` immediately -- non-retryable client errors.
   - **204**: Return `{ data: null as T, status: 204 }` for successful deletes.
   - **200-299**: Parse JSON, return `{ data, status }`.

**Retry config** (Claude's discretion, informed by research):
```
maxRetries: 3
baseDelayMs: 1000
maxDelayMs: 30000
retryableStatuses: [429, 500, 502, 503, 504]
```

**Backoff formula:** `min(baseDelayMs * 2^attempt + random(0, baseDelayMs), maxDelayMs)`

**Convenience methods:**
- `get<T>(path: string): Promise<ApiResponse<T>>` -- calls request("GET", path)
- `post<T>(path: string, body: unknown): Promise<ApiResponse<T>>` -- calls request("POST", path, body)
- `delete<T>(path: string): Promise<ApiResponse<T>>` -- calls request("DELETE", path)

**Private helper:**
- `sleep(ms: number): Promise<void>` -- promisified setTimeout

Export the class and the `ApiResponse<T>` interface as named exports.

CRITICAL: The API key must NEVER appear in error messages, logs, or thrown errors. Error messages should say "API key" not the actual key value.

Implement the `fetchWithFallback` function that wraps API calls with cache-first fallback behavior. This implements the locked decision: "Rate limit hit -> serve cached + inform. Never block the agent completely."

**Function signature:**
```typescript
async function fetchWithFallback<T>(
  cacheKey: string,
  fetcher: () => Promise<T>,
  cache: TtlCache,
  rateLimiter: TokenBucketRateLimiter,
  ttlMs: number,
  computeFields?: (data: T) => Record<string, unknown>,
): Promise<ToolResponse<T>>
```

**Logic flow:**
1. Try the fetcher function (which internally uses the API client).
2. On success:
   - Store result in cache with ttlMs
   - Return ToolResponse with data_source: "live", cache_age_seconds: 0, computed fields, api_calls_remaining from rateLimiter.getStatus()
3. On RateLimitError:
   - Check cache for key
   - If cached: return ToolResponse with data_source: "cached", cache_age_seconds from cache, and a note: "Using cached data (X min old). Fresh data available in Y minutes." (locked decision text)
   - If no cache: throw with message including retry timing
4. On other errors (ApiError, network errors):
   - Check cache for key
   - If cached: return cached data with note about the error
   - If no cache: re-throw the original error

The `computeFields` parameter is optional -- when provided, computed fields are calculated from the data (either live or cached). When not provided, `computed` is an empty object.

Every returned ToolResponse MUST include:
- `meta.cache_age_seconds` (0 for live, actual age for cached)
- `meta.data_source` ("live" or "cached")
- `meta.api_calls_remaining` (from rate limiter)
- `meta.fetched_at` (ISO timestamp)

Export the function as a named export.

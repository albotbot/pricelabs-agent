**Section 3: Common Mistakes** (from research/07-common-mistakes.md):
- 14 common mistakes STR hosts make with PriceLabs, organized as:
  - What the mistake is
  - Why hosts do it
  - Why it hurts revenue
  - How to detect it (warning signs the agent should watch for)
  - How to fix it
- Key DO NOT list items:
  - Do NOT set min price = base price (destroys dynamic range)
  - Do NOT ignore orphan days (7% revenue left on table)
  - Do NOT change base price weekly (creates volatility, confuse the algorithm)
  - Do NOT use "set and forget" approach (weekly monitoring essential)
  - Do NOT blindly copy competitor prices (different properties, different value props)

**Section 4: API Field Reference** (from research/02-api-reference.md):
- Non-obvious fields and their interpretation:
  - `recommended_base_price`: String, not number. Parse carefully
  - `demand_color`: Hex color code. Map to demand level
  - `demand_desc`: Human-readable demand level string
  - `booking_status` / `booking_status_STLY`: Compare for pace analysis
  - `ADR` / `ADR_STLY`: -1 means unavailable data, not zero
  - `unbookable`: "0" or "1" as string, not boolean
  - `last_date_pushed` / `last_refreshed_at`: ISO timestamps for sync health
  - `health_7_day`, `health_30_day`, `health_60_day`: String health indicators
- DSO Business Rules:
  - Percentage range: -75 to 500
  - Fixed price currency MUST match PMS currency
  - min_stay must be integer > 0
  - check_in/check_out: 7-char binary string (Mon-Sun, e.g., "1111100")
  - Erroneous dates silently omitted from response
  - DSO overrides ALL other settings including min price (highest priority)

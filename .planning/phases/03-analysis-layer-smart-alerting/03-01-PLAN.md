    **db/queries/analysis.ts:** Create prepared statement query module following the exact pattern from `db/queries/listing-snapshots.ts`:
    - Use `import * as BetterSqlite3 from "better-sqlite3"` (namespace import, per project decision)
    - Add eslint-disable comment for consistent-type-imports (same as other query modules)
    - Export `PortfolioKpiRow` interface with: listing_id, pms, name, occupancy_next_30, market_occupancy_next_30, revenue_past_7, stly_revenue_past_7, base_price, recommended_base_price, health_7_day, health_30_day, occupancy_gap_pct, revenue_vs_stly_pct, prev_occupancy_next_30, prev_revenue_past_7, prev_occupancy_gap_pct, prev_revenue_vs_stly_pct (all nullable except listing_id, pms)
    - Export `UnderperformerRow` interface with: listing_id, pms, name, occupancy_next_30, market_occupancy_next_30, occupancy_gap_pct, revenue_past_7, stly_revenue_past_7, revenue_vs_stly_pct, health_7_day, health_30_day, base_price, recommended_base_price, underperformance_type (all nullable except listing_id, pms, underperformance_type)
    - Export `MarketPositionRow` interface with: listing_id, pms, name, base_price, p25_price, p50_price, p75_price, p90_price, market_occupancy, listings_used, price_position (all nullable except listing_id, pms, price_position)
    - Export `createAnalysisQueries(db: BetterSqlite3.Database)` returning object with 3 prepared statements:

    1. `getPortfolioWoW`: Joins listing_snapshots current date with previous date (param-driven) for WoW comparison. See Research "Week-over-Week Listing Snapshot Comparison" SQL. Type params: `{ current_date: string; prev_date: string }`, returns `PortfolioKpiRow`.

    2. `getUnderperformers`: Selects from listing_snapshots WHERE any threshold exceeded (occupancy_gap_pct > @occ_gap_threshold OR revenue_vs_stly_pct < @revenue_stly_threshold OR both health scores < 50). Computes `underperformance_type` as CASE expression with priority ordering: 'occupancy_and_revenue' when both, then 'occupancy_gap', 'revenue_drop', 'health_decline', 'multiple'. ORDER BY severity then occupancy_gap_pct DESC. Type params: `{ snapshot_date: string; occ_gap_threshold: number; revenue_stly_threshold: number }`, returns `UnderperformerRow`.

    3. `getMarketPosition`: Joins listing_snapshots with market_snapshots on listing_id+pms, using MAX(snapshot_date) for each table. Computes `price_position` CASE: below_25th, 25th_to_50th, 50th_to_75th, 75th_to_90th, above_90th based on base_price vs percentile prices. Type params: `{}` (no params, uses latest data), returns `MarketPositionRow`.

    Use the exact SQL from Research sections "Week-over-Week", "Underperformer Detection", and "Market Position Comparison" as the basis, with minor adjustments for the typed interface.

    **tools/analysis.ts:** Create MCP tool registration module following the exact pattern from `tools/monitoring.ts` and `tools/audit.ts`:
    - Import McpServer type, Database type (`import type Database from "better-sqlite3"`), schemas from `../schemas/analysis.js`, and createAnalysisQueries from `../db/queries/analysis.js`
    - Export `registerAnalysisTools(server: McpServer, db: Database.Database): void`
    - Initialize queries with `const analysisQueries = createAnalysisQueries(db);`

    Register `pricelabs_get_portfolio_kpis`:
    - Description: "Compute portfolio KPIs (occupancy, revenue, health) with week-over-week and STLY comparisons for each listing. Returns per-listing data with current and previous period metrics. Use for weekly optimization reports (ANLY-03, ANLY-04) and competitive position analysis (ANLY-05)."
    - inputSchema: GetPortfolioKpisInputSchema.shape
    - annotations: readOnlyHint true, destructiveHint false, openWorldHint false
    - Handler logic:
      1. Determine `currentDate`: use params.current_date if provided, else query `SELECT MAX(snapshot_date) FROM listing_snapshots` using `db.prepare(...).pluck().get()` to get latest date as string
      2. Determine `prevDate` based on `params.compare_to` (default "previous_week"):
         - "previous_week": compute date 7 days before currentDate using `new Date(currentDate)` and setDate(-7), format as YYYY-MM-DD
         - "previous_month": 30 days before
         - "stly": 365 days before
      3. Call `analysisQueries.getPortfolioWoW.all({ current_date: currentDate, prev_date: prevDate })`
      4. Call `analysisQueries.getMarketPosition.all({})` for competitive positioning
      5. Return JSON with `{ as_of: currentDate, compare_to: params.compare_to ?? "previous_week", prev_date: prevDate, listings: kpiRows, market_position: marketRows }`
      6. Wrap in standard try/catch returning `{ error: message }` with `isError: true` on failure (same pattern as monitoring.ts)

    Register `pricelabs_detect_underperformers`:
    - Description: "Detect underperforming listings based on configurable thresholds for occupancy gap vs market, revenue vs STLY, and health score decline. Returns flagged listings with underperformance type classification. Use for underperformance alerting (ANLY-01, ANLY-02)."
    - inputSchema: DetectUnderperformersInputSchema.shape
    - annotations: readOnlyHint true, destructiveHint false, openWorldHint false
    - Handler logic:
      1. Determine `snapshotDate`: use params.snapshot_date if provided, else query MAX(snapshot_date) from listing_snapshots (same pattern as above)
      2. Apply defaults: `occGapThreshold = params.occupancy_gap_threshold ?? 20`, `revenueStlyThreshold = params.revenue_stly_threshold ?? -25`
      3. Call `analysisQueries.getUnderperformers.all({ snapshot_date: snapshotDate, occ_gap_threshold: occGapThreshold, revenue_stly_threshold: revenueStlyThreshold })`
      4. For each underperformer, also get market position by finding the matching row from `analysisQueries.getMarketPosition.all({})` to enrich with competitive context
      5. Return JSON with `{ as_of: snapshotDate, thresholds: { occupancy_gap_pct: occGapThreshold, revenue_vs_stly_pct: revenueStlyThreshold, health_score: 50 }, underperformers: enrichedRows, count: enrichedRows.length }`
      6. Standard try/catch error pattern

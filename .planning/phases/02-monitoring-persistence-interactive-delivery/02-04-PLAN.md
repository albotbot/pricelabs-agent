    **Create `src/tools/monitoring.ts`** exporting `registerMonitoringTools(server, db)`:

    Accepts `server: McpServer` and `db: Database.Database`.

    Create reservation queries: `const reservationQueries = createReservationQueries(db)`

    Register 1 tool:

    **pricelabs_get_booking_pace**
    - Description: "Calculate booking pace at multiple cutoff windows (7/30/60/90 days) compared to same-time-last-year (STLY). Use for MON-03 (pace tracking) and MON-04 (pace alert detection)."
    - inputSchema: `GetBookingPaceInputSchema.shape`
    - Annotations: `{ readOnlyHint: true, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Default cutoff_days to `[7, 30, 60, 90]` if not provided
      2. Get today as YYYY-MM-DD string
      3. For each cutoff in cutoff_days:
         a. Calculate end_date = today
         b. Calculate start_date by subtracting cutoff days from today
         c. Call `reservationQueries.getBookingPace({ listing_id, pms, start_date, end_date })`
         d. Call `reservationQueries.getStlyPace({ listing_id, pms, start_date, end_date })`
         e. Compute pace_vs_stly_pct: `((current.total_revenue - stly.total_revenue) / stly.total_revenue) * 100` (null-safe)
         f. Compute is_behind_stly: `pace_vs_stly_pct < -20` (configurable threshold, default -20%)
      4. Return JSON with array of pace objects: `{ cutoff_days, current: { bookings, booked_nights, total_revenue }, stly: { bookings, booked_nights, total_revenue }, pace_vs_stly_pct, is_behind_stly }`

    **Create `src/tools/audit.ts`** exporting `registerAuditTools(server, db)`:

    Accepts `server: McpServer` and `db: Database.Database`.

    Create audit queries: `const auditQueries = createAuditLogQueries(db)`

    Register 2 tools:

    **pricelabs_log_action**
    - Description: "Record an agent action in the audit log. Call this after every significant action: generating reports, sending alerts, making recommendations, receiving approvals, executing changes."
    - inputSchema: `LogActionInputSchema.shape`
    - Annotations: `{ readOnlyHint: false, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Call `auditQueries.insertEntry({ action_type, listing_id, pms, description, details_json, channel })`
      2. Return JSON with `{ logged: true, action_type, description }`

    **pricelabs_get_audit_log**
    - Description: "Retrieve audit trail entries. Filter by listing, action type, or date range. Use for accountability, debugging, and answering 'what changes have been made?' questions."
    - inputSchema: `GetAuditLogInputSchema.shape`
    - Annotations: `{ readOnlyHint: true, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Default start_date to 7 days ago (ISO datetime), end_date to now, limit to 50
      2. If listing_id provided, call `auditQueries.getByListing({ listing_id, limit })`
      3. Else if action_type provided, call `auditQueries.getByType({ action_type, limit })`
      4. Else call `auditQueries.getByDateRange({ start_date, end_date, limit })`
      5. Return JSON with `{ entries: result, count: result.length }`

    Both files follow Phase 1 patterns:
    - `type { McpServer }` import from `@modelcontextprotocol/sdk/server/mcp.js`
    - `type Database` import from `better-sqlite3`
    - Return format: `{ content: [{ type: "text" as const, text: JSON.stringify(result, null, 2) }] }`
    - Error handling: try/catch with `{ isError: true }` return

  <action>
    Create `src/tools/snapshots.ts` exporting a `registerSnapshotTools` function. This function accepts: `server: McpServer`, `db: Database.Database`.

    Inside the function, create all needed query instances by calling the factory functions from Plan 02's query modules:
    - `const listingQueries = createListingSnapshotQueries(db)`
    - `const priceQueries = createPriceSnapshotQueries(db)`
    - `const reservationQueries = createReservationQueries(db)`
    - `const marketQueries = createMarketSnapshotQueries(db)`

    Register 5 tools using `server.registerTool()` following the Phase 1 pattern:

    **Tool 1: pricelabs_store_daily_snapshots**
    - Description: "Store today's listing snapshots in the database for historical tracking. Call this during daily health checks after fetching all listings via pricelabs_get_listings."
    - inputSchema: `StoreDailySnapshotsInputSchema.shape`
    - Annotations: `{ readOnlyHint: false, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Determine snapshot_date: use params.snapshot_date if provided, else `new Date().toISOString().slice(0, 10)`
      2. Map each snapshot in params.snapshots to add snapshot_date and convert is_stale_sync boolean to 0/1 integer
      3. Call `listingQueries.insertMany(mappedSnapshots)`
      4. Return JSON with `{ stored: count, snapshot_date, message }` in `content[0].text`

    **Tool 2: pricelabs_store_price_snapshots**
    - Description: "Store price data with demand signals for a listing. Call this during daily health checks after fetching prices via pricelabs_get_prices."
    - inputSchema: `StorePriceSnapshotsInputSchema.shape`
    - Annotations: `{ readOnlyHint: false, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Determine snapshot_date
      2. Map prices to include listing_id, pms, snapshot_date
      3. Call `priceQueries.insertMany(mappedPrices)`
      4. Return JSON with `{ stored: count, listing_id, snapshot_date }`

    **Tool 3: pricelabs_store_reservations**
    - Description: "Upsert reservation data for a listing. Detects new cancellations by comparing booking_status changes. Call after fetching reservation data via pricelabs_get_reservations."
    - inputSchema: `StoreReservationsInputSchema.shape`
    - Annotations: `{ readOnlyHint: false, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Set today = `new Date().toISOString().slice(0, 10)`
      2. Map reservations to include listing_id, pms, first_seen_date=today, last_seen_date=today
      3. Call `reservationQueries.upsertMany(mappedReservations)`
      4. After upsert, query `reservationQueries.getRecentCancellations({ since: today })` to detect newly cancelled reservations
      5. Return JSON with `{ upserted: count, new_cancellations: [...] }`

    **Tool 4: pricelabs_get_snapshots**
    - Description: "Retrieve historical snapshots for trend analysis. Query listing, price, or market snapshot tables by date range. Use for comparing current performance to past periods."
    - inputSchema: `GetSnapshotsInputSchema.shape`
    - Annotations: `{ readOnlyHint: true, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Default start_date to 30 days ago, end_date to today, limit to 100
      2. Based on params.table:
         - "listing_snapshots": If listing_id provided, call `listingQueries.getSnapshotRange(...)`. Else call `listingQueries.getLatestForAllListings()`
         - "price_snapshots": Call `priceQueries.getByListingAndDateRange(...)` (requires listing_id)
         - "market_snapshots": Call `marketQueries.getSnapshotRange(...)` (requires listing_id)
      3. Return JSON with `{ table, rows: result, count: result.length }`

    **Tool 5: pricelabs_store_market_snapshot**
    - Description: "Store neighborhood/market data snapshots. Call after fetching neighborhood data via pricelabs_get_neighborhood for each listing."
    - inputSchema: `StoreMarketSnapshotInputSchema.shape`
    - Annotations: `{ readOnlyHint: false, destructiveHint: false, openWorldHint: false }`
    - Handler:
      1. Determine snapshot_date
      2. Map snapshots to include snapshot_date
      3. Call `marketQueries.insertMany(mappedSnapshots)`
      4. Return JSON with `{ stored: count, snapshot_date }`

    All handlers follow the Phase 1 return pattern:
    ```typescript
    return {
      content: [{ type: "text" as const, text: JSON.stringify(result, null, 2) }],
    };
    ```

    Error handling: Wrap each handler body in try/catch. On error, return `{ content: [{ type: "text", text: JSON.stringify({ error: error.message }) }], isError: true }`.

    Imports needed:
    - `type { McpServer }` from `@modelcontextprotocol/sdk/server/mcp.js`
    - `type Database` from `better-sqlite3`
    - All 5 Zod schemas from `../schemas/snapshots.js`
    - All 4 query factory functions from `../db/queries/*.js`

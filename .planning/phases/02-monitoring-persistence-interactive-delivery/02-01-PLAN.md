  <action>
    1. Install dependencies:
       ```bash
       cd mcp-servers/pricelabs
       npm install better-sqlite3
       npm install -D @types/better-sqlite3
       ```

    2. Create `src/services/database.ts` with an `initializeDatabase` function:
       - Accept optional `dbPath` parameter (string)
       - Default path: `process.env.PRICELABS_DB_PATH || path.join(process.env.HOME || "/tmp", ".pricelabs-agent", "data.sqlite")`
       - Create parent directory with `mkdirSync(dir, { recursive: true })` before opening
       - Import `Database` from `better-sqlite3` (default import)
       - Open database with `new Database(resolvedPath)`
       - Set pragmas in this order:
         a. `journal_mode = WAL` (concurrent reads + writes)
         b. `busy_timeout = 5000` (wait 5s for locks)
         c. `foreign_keys = ON` (referential integrity)
         d. `synchronous = NORMAL` (safe with WAL mode)
       - Return the `Database.Database` instance
       - Use `import type Database from "better-sqlite3"` for type-only imports where needed
       - Use `.js` extension on all local imports (ESM compatibility, matching Phase 1 convention)

    Do NOT export a singleton instance -- the caller (index.ts) will create and manage the instance. This keeps testability and avoids module-level side effects.
  </action>
  <verify>
    - `npm ls better-sqlite3` shows installed version
    - `npx tsc --noEmit` compiles without errors
    - `database.ts` exports `initializeDatabase` function
  </verify>
  <done>
    better-sqlite3 is installed in package.json (runtime dep) with @types/better-sqlite3 (dev dep). database.ts exports initializeDatabase that creates a WAL-mode SQLite database at a configurable path.

    Define a `Migration` interface: `{ version: number; description: string; up: (db: Database.Database) => void; }`

    **Migration 1 (version 1): listing_snapshots** (PERS-01)

    ```sql
    CREATE TABLE listing_snapshots (

      listing_id TEXT NOT NULL,
      pms TEXT NOT NULL,
      snapshot_date TEXT NOT NULL,
      name TEXT,
      health_7_day TEXT,
      health_30_day TEXT,

      occupancy_next_30 REAL,

      occupancy_gap_pct REAL,

      days_since_sync REAL,
      is_stale_sync INTEGER,
      data_json TEXT NOT NULL,
      created_at TEXT NOT NULL DEFAULT (datetime('now')),
      UNIQUE(listing_id, pms, snapshot_date)
    );
    CREATE INDEX idx_ls_date ON listing_snapshots(snapshot_date);
    CREATE INDEX idx_ls_listing ON listing_snapshots(listing_id, pms);
    ```

    **Migration 2 (version 2): price_snapshots** (PERS-02)
    ```sql
    CREATE TABLE price_snapshots (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      listing_id TEXT NOT NULL,

      snapshot_date TEXT NOT NULL,

    CREATE INDEX idx_ps_snapshot ON price_snapshots(snapshot_date);
    ```

    **Migration 3 (version 3): reservations** (PERS-03)
    ```sql
    CREATE TABLE reservations (
      id INTEGER PRIMARY KEY AUTOINCREMENT,

      pms TEXT NOT NULL,

    CREATE INDEX idx_res_listing ON reservations(listing_id, pms);
    CREATE INDEX idx_res_checkin ON reservations(check_in);
    CREATE INDEX idx_res_status ON reservations(booking_status);
    ```

    **Migration 4 (version 4): audit_log** (PERS-04)
    ```sql
    CREATE TABLE audit_log (
      id INTEGER PRIMARY KEY AUTOINCREMENT,

    CREATE INDEX idx_audit_listing ON audit_log(listing_id);
    CREATE INDEX idx_audit_type ON audit_log(action_type);
    ```

    **Migration 5 (version 5): market_snapshots** (PERS-05)
    ```sql
    CREATE TABLE market_snapshots (
      id INTEGER PRIMARY KEY AUTOINCREMENT,
      listing_id TEXT NOT NULL,
      pms TEXT NOT NULL,
      snapshot_date TEXT NOT NULL,

      p25_price REAL,

      created_at TEXT NOT NULL DEFAULT (datetime('now')),
      UNIQUE(listing_id, pms, snapshot_date)
    );
    CREATE INDEX idx_ms_listing_date ON market_snapshots(listing_id, pms, snapshot_date);
    ```

    The `runMigrations` function:
    1. Read current version: `db.pragma("user_version", { simple: true }) as number`
    2. Filter migrations where `version > currentVersion`
    3. For each pending migration, wrap in `db.transaction(() => { ... })()`
    4. Inside transaction: call `migration.up(db)` then `db.pragma(\`user_version = ${migration.version}\`)`
    5. Return early if no pending migrations

    Import `type Database from "better-sqlite3"` (type-only import). Use `.js` extensions for ESM.

    Create 5 query modules, each following the same pattern:
    - Import `type Database from "better-sqlite3"`
    - Export a factory function that accepts `db: Database.Database` and returns an object with prepared statements
    - Prepared statements use `db.prepare()` with named parameters (`@param_name`)
    - Batch inserts use `db.transaction()` for atomicity

    **listing-snapshots.ts** -- `createListingSnapshotQueries(db)`:
    - `insertSnapshot`: INSERT OR REPLACE into listing_snapshots with all columns from migration 1. Parameters: @listing_id, @pms, @snapshot_date, @name, @health_7_day, @health_30_day, @health_60_day, @occupancy_next_30, @market_occupancy_next_30, @occupancy_gap_pct, @revenue_past_7, @stly_revenue_past_7, @revenue_vs_stly_pct, @base_price, @recommended_base_price, @last_date_pushed, @days_since_sync, @is_stale_sync, @data_json
    - `getLatestSnapshot`: SELECT * WHERE listing_id = @listing_id AND pms = @pms ORDER BY snapshot_date DESC LIMIT 1
    - `getSnapshotRange`: SELECT * WHERE listing_id = @listing_id AND pms = @pms AND snapshot_date BETWEEN @start_date AND @end_date ORDER BY snapshot_date ASC
    - `getLatestForAllListings`: SELECT * WHERE snapshot_date = (SELECT MAX(snapshot_date) FROM listing_snapshots) -- latest day for all listings
    - `insertMany`: Transaction wrapping loop over `insertSnapshot.run()` -- accepts array of snapshot objects

    **price-snapshots.ts** -- `createPriceSnapshotQueries(db)`:
    - `insertSnapshot`: INSERT OR REPLACE with all columns from migration 2
    - `getByListingAndDateRange`: SELECT * WHERE listing_id = @listing_id AND pms = @pms AND price_date BETWEEN @start_date AND @end_date AND snapshot_date = @snapshot_date
    - `getLatestByListing`: SELECT * WHERE listing_id = @listing_id AND pms = @pms AND snapshot_date = (SELECT MAX(snapshot_date) FROM price_snapshots WHERE listing_id = @listing_id AND pms = @pms)
    - `insertMany`: Transaction wrapping loop over `insertSnapshot.run()`

    **reservations.ts** -- `createReservationQueries(db)`:
    - `upsertReservation`: INSERT INTO reservations (...) VALUES (...) ON CONFLICT(listing_id, pms, reservation_id) DO UPDATE SET check_in=excluded.check_in, check_out=excluded.check_out, booked_date=excluded.booked_date, booking_status=excluded.booking_status, rental_revenue=excluded.rental_revenue, total_cost=excluded.total_cost, no_of_days=excluded.no_of_days, currency=excluded.currency, last_seen_date=excluded.last_seen_date, data_json=excluded.data_json, updated_at=datetime('now'). Also detect cancellations: set cancelled_on=datetime('now') when booking_status changes to 'cancelled'.
    - `getByListing`: SELECT * WHERE listing_id = @listing_id AND pms = @pms ORDER BY check_in ASC
    - `getBookingPace`: COUNT bookings with check_in BETWEEN @start_date AND @end_date, booking_status = 'booked', for a given listing_id + pms. Return count, total booked_nights (SUM of no_of_days), total_revenue (SUM of rental_revenue).
    - `getStlyPace`: Same as getBookingPace but date-shift: check_in BETWEEN date(@start_date, '-1 year') AND date(@end_date, '-1 year')
    - `getRecentCancellations`: SELECT * WHERE booking_status = 'cancelled' AND updated_at > @since ORDER BY updated_at DESC
    - `upsertMany`: Transaction wrapping loop over `upsertReservation.run()`

    **audit-log.ts** -- `createAuditLogQueries(db)`:
    - `insertEntry`: INSERT INTO audit_log (action_type, listing_id, pms, description, details_json, channel) VALUES (...)
    - `getByDateRange`: SELECT * WHERE created_at BETWEEN @start_date AND @end_date ORDER BY created_at DESC LIMIT @limit
    - `getByListing`: SELECT * WHERE listing_id = @listing_id ORDER BY created_at DESC LIMIT @limit
    - `getByType`: SELECT * WHERE action_type = @action_type ORDER BY created_at DESC LIMIT @limit
    - `getLatestAlert`: SELECT * WHERE action_type = 'alert' AND listing_id = @listing_id ORDER BY created_at DESC LIMIT 1 -- for dedup (Pitfall 6)

    **market-snapshots.ts** -- `createMarketSnapshotQueries(db)`:
    - `insertSnapshot`: INSERT OR REPLACE with all columns from migration 5
    - `getLatestByListing`: SELECT * WHERE listing_id = @listing_id AND pms = @pms ORDER BY snapshot_date DESC LIMIT 1
    - `getSnapshotRange`: SELECT * WHERE listing_id = @listing_id AND pms = @pms AND snapshot_date BETWEEN @start_date AND @end_date
    - `insertMany`: Transaction wrapping loop

    IMPORTANT: All query modules use `.js` extension on imports for ESM. Use `import type Database from "better-sqlite3"` (type-only).

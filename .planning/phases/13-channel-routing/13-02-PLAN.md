    PREREQUISITE: The user must have already created the #pricelabs Slack channel and invited the bot to it. The Slack channel ID (starts with "C") is needed.

    1. Obtain the #pricelabs Slack channel ID. Ask the user if not already known. The channel ID looks like "C0XXXXXXXX". Can be found by right-clicking the channel in Slack > "Copy link" > extract the ID from the URL, or the user can provide it directly.

    2. Back up openclaw.json:
       `cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak`

    3. Read openclaw.json. Add the #pricelabs channel to the Slack channel allowlist at `channels.slack.channels`:
       ```json
       "C_PRICELABS_CHANNEL_ID": {
         "allow": true,
         "requireMention": false,
         "allowBots": true
       }
       ```
       Replace `C_PRICELABS_CHANNEL_ID` with the actual channel ID obtained in step 1.

       Key: `requireMention: false` means Prism responds to every message in the dedicated channel without needing @-mention. This is correct UX for a dedicated agent channel.

       Do NOT modify existing channel entries (C0AF9MXD0ER, C0AG7FJNKNC or any others).

    4. Add a Slack peer-channel binding to the existing `bindings[]` array (created in Plan 01):
       ```json
       {
         "agentId": "pricelabs",
         "match": {
           "channel": "slack",
           "peer": {
             "kind": "channel",
             "id": "C_PRICELABS_CHANNEL_ID"
           }
         }
       }
       ```
       Replace `C_PRICELABS_CHANNEL_ID` with the actual channel ID.
       APPEND to the existing bindings array -- do NOT replace the Telegram binding from Plan 01.

    5. Restart gateway:
       `sudo systemctl restart openclaw-gateway.service`

    6. Wait for cooldown:
       `sleep 30`

    7. Verify channels and routing:
       `openclaw channels status --probe`
       `openclaw agents list --bindings`
       Expected: Slack connected, pricelabs agent has both Telegram and Slack bindings.

    IMPORTANT: Use the ACTUAL Slack channel ID, not a placeholder. If the channel has not been created yet, pause and ask the user to create it first and provide the channel ID.

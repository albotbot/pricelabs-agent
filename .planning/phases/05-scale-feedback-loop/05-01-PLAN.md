**Migration 7 (user_config):**
```sql
CREATE TABLE user_config (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  config_key TEXT NOT NULL,
  config_value TEXT NOT NULL,
  listing_id TEXT,
  pms TEXT,
  listing_id_key TEXT GENERATED ALWAYS AS (COALESCE(listing_id, '__global__')) STORED,
  pms_key TEXT GENERATED ALWAYS AS (COALESCE(pms, '__global__')) STORED,
  updated_at TEXT NOT NULL DEFAULT (datetime('now')),
  UNIQUE(config_key, listing_id_key, pms_key)
);
CREATE INDEX idx_uc_key ON user_config(config_key);
CREATE INDEX idx_uc_listing ON user_config(listing_id, pms);
```

The generated stored columns (`listing_id_key`, `pms_key`) use COALESCE to map NULLs to the sentinel value `'__global__'`. This is necessary because SQLite treats NULL as distinct for UNIQUE purposes, so without it multiple global rows with the same config_key would be allowed. SQLite prohibits expressions directly in UNIQUE constraints, so the generated columns serve as the workaround.

Follow the existing migration pattern: `{ version: N, description: "...", up(db) { db.exec(\`...\`); } }`.

Export `createChangeTrackingQueries(db: BetterSqlite3.Database)` returning:

1. `insertTracking` -- INSERT into change_tracking with all fields. Parameters: `@audit_log_id`, `@listing_id`, `@pms`, `@change_type`, `@change_date`, `@affected_dates_start`, `@affected_dates_end`, `@before_json`, `@after_json`, `@check_7d_due`, `@check_14d_due`, `@check_30d_due`.

2. `getPendingChecks` -- SELECT * from change_tracking WHERE any of the three check intervals is due (done=0 AND due <= @today). Order by change_date ASC.

3. `markCheckDone` -- UPDATE change_tracking using CASE expressions to set the correct interval's done flag and result_json based on @interval (7, 14, or 30) parameter. WHERE id = @id.

4. `getByListing` -- SELECT * from change_tracking WHERE listing_id = @listing_id AND pms = @pms, ORDER BY change_date DESC, LIMIT @limit.

Define TypeScript row interface `ChangeTrackingRow` matching the table columns.

**`mcp-servers/pricelabs/src/db/queries/user-config.ts`:**

Export `createUserConfigQueries(db: BetterSqlite3.Database)` returning:

1. `getConfigValue` -- SELECT config_value from user_config WHERE config_key = @config_key AND ((listing_id = @listing_id AND pms = @pms) OR (listing_id IS NULL AND pms IS NULL)). ORDER BY CASE WHEN listing_id IS NOT NULL THEN 0 ELSE 1 END. LIMIT 1. This returns per-listing value with fallback to global.

2. `getAllForListing` -- SELECT with LEFT JOIN to merge per-listing overrides with global defaults. Parameters: @listing_id, @pms. Returns config_key, config_value, source ('listing' or 'global').

3. `getAllGlobal` -- SELECT config_key, config_value, updated_at from user_config WHERE listing_id IS NULL AND pms IS NULL, ORDER BY config_key ASC.

4. `upsertConfig` -- INSERT with ON CONFLICT(config_key, listing_id_key, pms_key) DO UPDATE SET config_value and updated_at. Parameters: @config_key, @config_value, @listing_id, @pms. Note: The ON CONFLICT clause references the generated stored column names, not the raw listing_id/pms columns.

5. `deleteListingOverride` -- DELETE FROM user_config WHERE config_key = @config_key AND listing_id = @listing_id AND pms = @pms.

Define TypeScript row interfaces: `UserConfigRow` (full row), `UserConfigValueRow` (just config_value from getConfigValue), `UserConfigMergedRow` (config_key, config_value, source from getAllForListing).
